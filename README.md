# BOTA

אתר React (Vite) עם שני דפים נפרדים, ופריסה אוטומטית ל-GitHub Pages דרך GitHub Actions:

- **האתר הציבורי** (`/`) — כניסה לפי מספר טלפון, רשימת אפליקציות, הורדה, בקשות גישה, פינת שותפות, צ'אט AI.
- **פאנל הניהול** (`/admin.html`) — דף נפרד לגמרי, מוגן בקוד גישה, ובו כל כלי הניהול (פרסום/עריכה/מחיקה, אישור גישות, בקשות עלייה, הגדרות).

## איך מעלים ל-GitHub

1. תיצור ריפו חדש (ריק, בלי README) ב-GitHub.
2. תעלה את כל הקבצים מהתיקייה הזו לריפו.
3. בריפו: **Settings → Pages → Build and deployment → Source** → תבחר **GitHub Actions**.
4. כל push ל-`main` יבנה ויפרסם אוטומטית.
5. הכתובות הסופיות:
   - אתר ציבורי: `https://<username>.github.io/<repo-name>/`
   - פאנל ניהול: `https://<username>.github.io/<repo-name>/admin.html`

## קוד הגישה לפאנל הניהול

קוד הגישה הנוכחי: **Bota app2026**

הקוד **לא כתוב בגלוי** בקוד המקור — בקובץ `src/AdminApp.jsx` שמור רק ה-hash שלו (SHA-256), והבדיקה מתבצעת מול ה-hash. מי שיפתח את הקוד/את הבאנדל בדפדפן לא יראה את הסיסמה עצמה, רק מחרוזת hash חסרת משמעות.

**חשוב להבין את המגבלה:** זו הגנה סבירה לפרויקט אישי, אבל היא לא "הצפנה" במובן החזק — קוד גישה קצר, גם אם מגובב (hashed), עדיין אפשרי לפצח אם מישהו ינסה שילובים רבים (brute-force) על ה-hash שחשוף בקוד. זה לא מגן כמו סיסמה בשרת אמיתי, אבל מונע מכל מי שנכנס בטעות/סקרנות לראות את הקוד בפשטות.

### איך משנים את קוד הגישה

1. תריץ (בכל מחשב עם Node, או תבקש ממני) את הפקודה הבאה כדי לקבל hash חדש למילה/קוד שתבחר:
   ```
   node -e "const c=require('crypto');console.log(c.createHash('sha256').update('הקוד-החדש-שלך','utf8').digest('hex'))"
   ```
2. תעתיק את הפלט (מחרוזת של 64 תווים).
3. בקובץ `src/AdminApp.jsx`, תחליף את הערך של `ADMIN_HASH` בפלט החדש.
4. תעלה את השינוי ל-GitHub — זהו.

## מסד נתונים משותף (חנות אפליקציות אמיתית בין כל המשתמשים)

האפליקציות (שם, תיאור, קישור לפוסטר, קישור להורדה) נשמרות בקובץ `data/items.json` בתוך הריפו עצמו. כך כל מי שנכנס לאתר הציבורי — מכל מכשיר — רואה את אותן אפליקציות בדיוק.

### איך מגדירים

1. ב-GitHub: **Settings (של המשתמש שלך) → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token**.
2. תגביל אותו:
   - **Repository access** → **Only select repositories** → תבחר רק את הריפו הזה.
   - **Permissions** → **Contents** → **Read and write**. שאר ההרשאות "No access".
3. תעתיק את הטוקן (מוצג פעם אחת בלבד).
4. ב-`/admin.html` → תיכנס עם קוד הגישה → **הגדרות → מסד נתונים משותף (GitHub)** → תמלא Owner / Repo / Branch / Token → **בדוק חיבור** → **שמור הגדרות**.

מהרגע הזה, כל פעולת ניהול (פרסום/עריכה/מחיקה/אישור גישה) נשמרת גם בקובץ ב-GitHub ומתעדכנת אצל **כל** המשתמשים (יכול לקחת עד כמה דקות בגלל cache).

## נקודות חשובות

- **קוד הגישה נשמר רק בזיכרון הסשן של הדפדפן** (`sessionStorage`) — נדרש להזין אותו מחדש בכל פתיחה חדשה של הדפדפן/טאב.
- **טוקן ה-GitHub נשמר רק בדפדפן שממנו האדמין נכנס** (localStorage) — לא נכנס לקוד המקור ולא מגיע למשתמשים באתר הציבורי.
- **בקשות גישה ובקשות עלייה מהקהילה** נשמרות מקומית (localStorage) בלבד ולא מסונכרנות ל-GitHub. אם משתמש שולח בקשה ממכשיר אחד, האדמין יראה אותה רק אם ייכנס לפאנל הניהול מאותו דפדפן/מכשיר. לשיתוף מלא בין מכשירים גם בבקשות אלה נדרש שרת קטן.
- **תמונות פוסטר וקבצי הורדה** — עדיף להשתמש בשדות ה"קישור" (URL) ולא בהעלאת קובץ מהגלריה. העלאה הופכת את התוכן ל-base64 שנכנס ישירות ל-`items.json`, מה שינפח את הקובץ. קישור חיצוני (imgur לתמונות, Google Drive/Mega לקבצים) קליל ומהיר בהרבה.
- **צ'אט ה-AI לא יעבוד כמו שהוא.** הקוד קורא ל-`api.anthropic.com` ישירות מהדפדפן — זה חסום מטעמי אבטחה/CORS באתר סטטי. כדי שזה יעבוד צריך שרת ביניים קטן (proxy) שמחזיק מפתח API בצד שרת — אם תרצה, אני יכול לבנות לך את זה בנפרד.
